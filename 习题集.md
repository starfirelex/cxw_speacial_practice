### 1.1 SQL概述（对应讲义1.1节）

#### ⭐ 基础题1：SQL基础概念

**题目**：请判断以下说法是否正确，并说明理由。

1. SQL是一种用于管理关系数据库系统的标准化查询语言。
2. SQL于1980年由Boyce和Chamberlin首次提出。
3. SQL在1986年被ANSI确立为标准。
4. SQL只能在MySQL数据库中使用。
5. SQL是声明式编程语言。

**参考答案**：
1. ✅ 正确。SQL（Structured Query Language）是用于管理关系数据库系统的标准化查询和程序设计语言。
2. ❌ 错误。SQL于1974年由Boyce和Chamberlin首次提出，不是1980年。
3. ✅ 正确。SQL在1986年被ANSI（美国国家标准协会）首次确立为标准。
4. ❌ 错误。SQL是标准语言，可以在多种关系型数据库中使用，如MySQL、Oracle、SQL Server、PostgreSQL等。
5. ✅ 正确。SQL是声明式编程语言，用户只需要指定"要什么"，而不需要指定"怎么做"。

**对应讲义**：第一章 1.1 SQL概述

#### ⭐ 基础题4：约束类型识别

**题目**：请说明以下约束的作用：
1. PRIMARY KEY
2. FOREIGN KEY
3. UNIQUE
4. NOT NULL
5. CHECK
6. DEFAULT

**参考答案**：
1. PRIMARY KEY（主键约束）：唯一标识表中的一条记录，不能重复且不能为空。
2. FOREIGN KEY（外键约束）：建立表之间的关联，保证数据引用完整性。
3. UNIQUE（唯一约束）：确保列中的值唯一，但允许为空（只能有一个NULL）。
4. NOT NULL（非空约束）：确保列中的值不能为空。
5. CHECK（检查约束）：限制列值的范围，满足指定条件。
6. DEFAULT（默认值约束）：为列设置默认值，插入数据时未指定则使用默认值。

#### ⭐⭐ 进阶题5：创建员工表（对应案例1）

**题目**：创建一个员工表（employees），包含以下字段和约束：
- id：主键，自增
- name：员工姓名，不能为空，长度50
- age：年龄，必须大于等于18
- gender：性别，只能是'男'或'女'
- salary：薪资，默认值为5000.00
- department：部门名称，不能为空
- hire_date：入职日期

**参考答案**：
```sql
CREATE TABLE employees (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    age INT CHECK (age >= 18),
    gender VARCHAR(10) CHECK (gender IN ('男', '女')),
    salary DECIMAL(10,2) DEFAULT 5000.00,
    department VARCHAR(50) NOT NULL,
    hire_date DATE
);
```

**解析**：
- 使用`PRIMARY KEY AUTO_INCREMENT`设置主键自增
- 使用`NOT NULL`确保name和department不能为空
- 使用`CHECK (age >= 18)`限制年龄必须大于等于18
- 使用`CHECK (gender IN ('男', '女'))`限制性别只能是'男'或'女'
- 使用`DEFAULT 5000.00`设置默认薪资

**对应讲义**：第一章 1.2.1 DDL - 案例1：学生信息管理系统

#### ⭐⭐⭐ 实战题7：设计图书管理系统数据库

**题目**：设计一个图书管理系统的数据库结构，包含以下实体：
- 书籍（books）：book_id（主键）、title、author、price、stock（库存）
- 读者（readers）：reader_id（主键）、name、phone、register_date（注册日期）
- 借阅记录（borrow_records）：record_id（主键）、book_id（外键）、reader_id（外键）、borrow_date（借阅日期）、return_date（归还日期，可为空）

要求：
1. 创建三个表
2. 设置适当的主键和外键
3. 设置必要的约束

**参考答案**：
```sql
-- 创建书籍表
CREATE TABLE books (
    book_id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(100) NOT NULL,
    author VARCHAR(50) NOT NULL,
    price DECIMAL(10,2) CHECK (price > 0),
    stock INT DEFAULT 0 CHECK (stock >= 0)
);

-- 创建读者表
CREATE TABLE readers (
    reader_id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    phone VARCHAR(20),
    register_date DATE NOT NULL
);

-- 创建借阅记录表
CREATE TABLE borrow_records (
    record_id INT PRIMARY KEY AUTO_INCREMENT,
    book_id INT NOT NULL,
    reader_id INT NOT NULL,
    borrow_date DATE NOT NULL,
    return_date DATE,
    FOREIGN KEY (book_id) REFERENCES books(book_id),
    FOREIGN KEY (reader_id) REFERENCES readers(reader_id)
);
```

**解析**：
- 三个表分别对应三个实体
- 使用AUTO_INCREMENT设置主键自增
- 使用FOREIGN KEY建立表之间的关联
- 使用CHECK约束限制price必须大于0，stock必须大于等于0
- return_date允许为NULL，因为借书时归还日期还未确定

**对应讲义**：第一章 1.2.1 DDL - 外键约束、多表设计

**题目**：假设有students表，请写出以下操作的SQL语句：
1. 删除id为10的学生
2. 删除年龄小于18岁的学生
3. 删除姓'张'的学生
4. 删除所有数据（清空表）

**参考答案**：
```sql
-- 1. 删除id为10的学生
DELETE FROM students WHERE id = 10;

-- 2. 删除年龄小于18岁的学生
DELETE FROM students WHERE age < 18;

-- 3. 删除姓'张'的学生
DELETE FROM students WHERE name LIKE '张%';

-- 4. 删除所有数据

```

- LIKE运算符用于模糊匹配，'张%'表示以'张'开头的字符串
- TRUNCATE TABLE比DELETE更快，因为不记录日志，但不可回滚

#### ⭐⭐ 进阶题16：GROUP BY分组（对应案例3）

**题目**：假设有students表（id, name, age, gender, class），请写出以下查询：
1. 按性别统计学生人数
2. 按班级统计平均年龄
3. 按班级统计每个班级的学生人数
4. 按性别和班级统计学生人数
5. 查找每个班级的最高分（假设有score字段）

**参考答案**：
```sql
-- 1. 按性别统计学生人数
SELECT gender, COUNT(*) AS 人数 FROM students GROUP BY gender;

-- 2. 按班级统计平均年龄
SELECT class, AVG(age) AS 平均年龄 FROM students GROUP BY class;

-- 3. 按班级统计每个班级的学生人数
SELECT class, COUNT(*) AS 人数 FROM students GROUP BY class;

-- 4. 按性别和班级统计学生人数
SELECT gender, class, COUNT(*) AS 人数 
FROM students 
GROUP BY gender, class;

-- 5. 查找每个班级的最高分
SELECT class, MAX(score) AS 最高分 
FROM students 
GROUP BY class;
```

**解析**：
- GROUP BY按指定字段分组
- 聚合函数（COUNT、AVG、MAX等）对每组数据进行统计
- 可以按多个字段分组（性别和班级）
- 使用AS给结果列起别名，提高可读性

**对应讲义**：第一章 1.2.3 DQL - 案例3：学生成绩查询

#### ⭐⭐ 进阶题17：HAVING分组筛选

**题目**：假设有students表（id, name, age, gender, class, score），请写出以下查询：
1. 查找平均分大于80的班级
2. 查找人数大于20的班级
3. 查找平均年龄大于20的性别组
4. 查找最高分大于95的学生所在的班级

**参考答案**：
```sql
-- 1. 查找平均分大于80的班级
SELECT class, AVG(score) AS 平均分
FROM students
GROUP BY class
HAVING AVG(score) > 80;

-- 2. 查找人数大于20的班级
SELECT class, COUNT(*) AS 人数
FROM students
GROUP BY class
HAVING COUNT(*) > 20;

-- 3. 查找平均年龄大于20的性别组
SELECT gender, AVG(age) AS 平均年龄
FROM students
GROUP BY gender
HAVING AVG(age) > 20;

-- 4. 查找最高分大于95的学生所在的班级
SELECT class, MAX(score) AS 最高分
FROM students
GROUP BY class
HAVING MAX(score) > 95;
```

**解析**：
- HAVING用于对分组后的结果进行筛选
- WHERE在分组前筛选记录，HAVING在分组后筛选组
- HAVING中可以使用聚合函数（如AVG、COUNT、MAX）

**对应讲义**：第一章 1.2.3 DQL - HAVING分组筛选

#### ⭐⭐⭐ 实战题18：多表连接查询（对应案例3）

**题目**：假设有以下三个表：

**students表**：
- id（主键）
- name（姓名）
- class_id（班级ID，外键）

**classes表**：
- id（主键）
- class_name（班级名称）

**scores表**：
- id（主键）
- student_id（学生ID，外键）
- course_name（课程名称）
- score（成绩）

请写出以下查询：
1. 查询所有学生的姓名和班级名称
2. 查询所有学生的姓名、课程名称和成绩
3. 查询每个学生的姓名和总成绩
4. 查询每个班级的平均成绩
5. 查询没有成绩的学生姓名和班级

**参考答案**：
```sql
-- 1. 查询所有学生的姓名和班级名称
SELECT s.name, c.class_name
FROM students s
INNER JOIN classes c ON s.class_id = c.id;

-- 2. 查询所有学生的姓名、课程名称和成绩
SELECT s.name, sc.course_name, sc.score
FROM students s
INNER JOIN scores sc ON s.id = sc.student_id;

-- 3. 查询每个学生的姓名和总成绩
SELECT s.name, SUM(sc.score) AS 总成绩
FROM students s
INNER JOIN scores sc ON s.id = sc.student_id
GROUP BY s.name;

-- 4. 查询每个班级的平均成绩
SELECT c.class_name, AVG(sc.score) AS 平均成绩
FROM classes c
LEFT JOIN students s ON c.id = s.class_id
LEFT JOIN scores sc ON s.id = sc.student_id
GROUP BY c.class_name;

-- 5. 查询没有成绩的学生姓名和班级
```

#### ⭐⭐ 进阶题27：事务ACID特性（对应案例5）

**题目**：请说明事务的四个ACID特性，并给出一个银行转账的例子说明。

**参考答案**：

**ACID特性**：
1. **原子性（Atomicity）**：事务中的操作要么全部成功，要么全部失败。
2. **一致性（Consistency）**：事务执行前后，数据库保持一致状态。
3. **隔离性（Isolation）**：多个事务并发执行时，互不干扰。
4. **持久性（Durability）**：事务提交后，数据永久保存。

**银行转账例子**：
```sql
-- 开始事务
START TRANSACTION;

-- 账户A减少1000元
UPDATE accounts SET balance = balance - 1000 WHERE id = 1;

-- 账户B增加1000元
UPDATE accounts SET balance = balance + 1000 WHERE id = 2;

-- 提交事务
COMMIT;
```

**说明**：
- 如果第二条UPDATE失败，整个事务会回滚，账户A的金额不会减少（原子性）
- 转账前后，两个账户的总金额保持不变（一致性）
- 如果同时有多个转账操作，互不干扰（隔离性）
- 提交后，即使系统崩溃，转账数据也不会丢失（持久性）

**对应讲义**：第二章 2.3.3 存储引擎 - 案例5：MySQL存储引擎对比

**题目**：假设有一个订单系统，包含以下表：

**要求**：使用事务处理一个订单创建的过程，包括：

2. 创建订单明细
3. 扣减库存
4. 如果任何一步失败，整个操作回滚

**参考答案**：
```sql
-- 开始事务
START TRANSACTION;

-- 1. 创建订单
INSERT INTO orders (user_id, total_amount) 
VALUES (1, 299.00);
SET @order_id = LAST_INSERT_ID();

-- 2. 创建订单明细（购买商品1，数量1）
INSERT INTO order_items (order_id, product_id, quantity)
VALUES (@order_id, 1, 1);

-- 3. 扣减库存
UPDATE products 
SET stock = stock - 1 
WHERE id = 1 AND stock >= 1;

-- 检查库存是否充足
IF ROW_COUNT() = 0 THEN
    -- 库存不足，回滚事务
    ROLLBACK;
    SELECT '库存不足，订单创建失败' AS message;
ELSE
    -- 提交事务
    COMMIT;
    SELECT '订单创建成功' AS message;
END IF;
```

- LAST_INSERT_ID()获取最后插入的自增ID
- UPDATE时检查库存是否充足（stock >= 1）
- ROW_COUNT()检查影响的行数，如果为0说明库存不足
- 库存不足时使用ROLLBACK回滚
- 成功时使用COMMIT提交

```sql
    COALESCE(SUM(o.total_amount), 0) AS 总金额
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
GROUP BY u.id, u.username;
```

**4. 查询每个商品的销售数量**：
```sql
SELECT 
    p.name AS 商品名称,
    SUM(oi.quantity) AS 销售数量
FROM products p
LEFT JOIN order_items oi ON p.id = oi.product_id
GROUP BY p.id, p.name
ORDER BY 销售数量 DESC;
```

**5. 查询某用户的所有订单详情**：
```sql
SELECT 
    o.id AS 订单ID,
    o.order_date AS 下单时间,
    o.total_amount AS 订单金额,
    o.status AS 订单状态,
    GROUP_CONCAT(p.name, ' x', oi.quantity) AS 商品明细
FROM orders o
JOIN order_items oi ON o.id = oi.order_id
JOIN products p ON oi.product_id = p.id
WHERE o.user_id = 1
GROUP BY o.id, o.order_date, o.total_amount, o.status;
```

**6. 查询销量前10的商品**：
```sql
SELECT 
    p.name AS 商品名称,
    p.category AS 商品类别,
    SUM(oi.quantity) AS 销售数量,
    SUM(oi.quantity * oi.price) AS 销售总额
FROM products p
JOIN order_items oi ON p.id = oi.product_id
GROUP BY p.id, p.name, p.category
ORDER BY 销售数量 DESC
LIMIT 10;
```

**7. 使用事务处理订单创建过程**：
```sql
-- 创建订单的事务过程
START TRANSACTION;

-- 假设用户1购买商品1（笔记本电脑）1件，商品2（智能手机）1件

-- 1. 检查库存
SELECT stock FROM products WHERE id = 1;
SELECT stock FROM products WHERE id = 2;

-- 2. 创建订单
INSERT INTO orders (user_id, total_amount, status) 
VALUES (1, 5999.00 + 2999.00, 'completed');

-- 获取订单ID
SET @order_id = LAST_INSERT_ID();

-- 3. 创建订单明细
INSERT INTO order_items (order_id, product_id, quantity, price) VALUES
(@order_id, 1, 1, 5999.00),
(@order_id, 2, 1, 2999.00);

-- 4. 扣减库存
UPDATE products SET stock = stock - 1 WHERE id = 1;
UPDATE products SET stock = stock - 1 WHERE id = 2;

-- 5. 提交事务
COMMIT;
```

**解析**：
- 使用外键建立表之间的关联
- 使用LEFT JOIN确保所有用户都被统计
- 使用COALESCE处理NULL值
- 使用GROUP_CONCAT合并多个商品信息
- 使用事务保证订单创建的原子性

#### ⭐⭐⭐ 实战题32：图书借阅系统

**题目**：实现一个图书借阅系统的完整功能：

**数据库设计**：
1. books表（图书表）：id, title, author, isbn, category, stock
2. readers表（读者表）：id, name, phone, register_date
3. borrow_records表（借阅记录表）：id, book_id, reader_id, borrow_date, return_date, status

**功能要求**：
1. 创建数据库和表
2. 实现图书借阅功能（扣减库存，创建借阅记录）
3. 实现图书归还功能（增加库存，更新归还日期）
4. 查询当前借出的图书
5. 查询逾期未还的图书（假设借阅期限为30天）
6. 查询每个读者的借阅次数
7. 查询最受欢迎的图书（借阅次数最多的前10本）

**参考答案**：

**1. 创建数据库和表**：
```sql
CREATE DATABASE library;
USE library;

CREATE TABLE books (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(100) NOT NULL,
    author VARCHAR(50) NOT NULL,
    isbn VARCHAR(20) UNIQUE,
    category VARCHAR(50),
    stock INT DEFAULT 1 CHECK (stock >= 0)
);

CREATE TABLE readers (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    phone VARCHAR(20),
    register_date DATE
);

CREATE TABLE borrow_records (
    id INT PRIMARY KEY AUTO_INCREMENT,
    book_id INT NOT NULL,
    reader_id INT NOT NULL,
    borrow_date DATE NOT NULL,
    return_date DATE,
    status VARCHAR(20) DEFAULT 'borrowed',
    FOREIGN KEY (book_id) REFERENCES books(id),
    FOREIGN KEY (reader_id) REFERENCES readers(id)
);
```

**2. 图书借阅功能（使用事务）**：
```sql
START TRANSACTION;

-- 假设读者1借阅图书1

-- 1. 检查库存
SELECT stock FROM books WHERE id = 1;

-- 2. 检查是否已借阅
SELECT COUNT(*) FROM borrow_records 
WHERE book_id = 1 AND reader_id = 1 AND status = 'borrowed';

-- 3. 扣减库存
UPDATE books SET stock = stock - 1 WHERE id = 1 AND stock > 0;

-- 4. 创建借阅记录
INSERT INTO borrow_records (book_id, reader_id, borrow_date, status)
VALUES (1, 1, CURRENT_DATE, 'borrowed');

COMMIT;
```

**3. 图书归还功能（使用事务）**：
```sql
START TRANSACTION;

-- 假设读者1归还图书1，借阅记录ID为1

-- 1. 更新归还日期和状态
UPDATE borrow_records 
SET return_date = CURRENT_DATE, status = 'returned'
WHERE id = 1;

-- 2. 增加库存
UPDATE books SET stock = stock + 1 WHERE id = 1;

COMMIT;
```

**4. 查询当前借出的图书**：
```sql
SELECT 
    r.name AS 读者姓名,
    b.title AS 图书名称,
    b.author AS 作者,
    br.borrow_date AS 借阅日期
FROM borrow_records br
JOIN readers r ON br.reader_id = r.id
JOIN books b ON br.book_id = b.id
WHERE br.status = 'borrowed'
ORDER BY br.borrow_date DESC;
```

**5. 查询逾期未还的图书（借阅期限30天）**：
```sql
SELECT 
    r.name AS 读者姓名,
    r.phone AS 联系电话,
    b.title AS 图书名称,
    br.borrow_date AS 借阅日期,
    DATEDIFF(CURRENT_DATE, br.borrow_date) AS 已借阅天数,
    DATEDIFF(CURRENT_DATE, br.borrow_date) - 30 AS 逾期天数
FROM borrow_records br
JOIN readers r ON br.reader_id = r.id
JOIN books b ON br.book_id = b.id
WHERE br.status = 'borrowed'
  AND DATEDIFF(CURRENT_DATE, br.borrow_date) > 30
ORDER BY DATEDIFF(CURRENT_DATE, br.borrow_date) DESC;
```

**6. 查询每个读者的借阅次数**：
```sql
SELECT 
    r.name AS 读者姓名,
    COUNT(br.id) AS 借阅次数,
    COUNT(CASE WHEN br.status = 'borrowed' THEN 1 END) AS 当前借阅
FROM readers r
LEFT JOIN borrow_records br ON r.id = br.reader_id
GROUP BY r.id, r.name
ORDER BY 借阅次数 DESC;
```

**7. 查询最受欢迎的图书**：
```sql
SELECT 
    b.title AS 图书名称,
    b.author AS 作者,
    COUNT(br.id) AS 借阅次数,
    COUNT(DISTINCT br.reader_id) AS 借阅人数
FROM books b
LEFT JOIN borrow_records br ON b.id = br.book_id
GROUP BY b.id, b.title, b.author
ORDER BY 借阅次数 DESC
LIMIT 10;
```

**解析**：
- 使用DATEDIFF函数计算日期差
- 使用CASE WHEN统计当前借阅数量
- 使用COUNT(DISTINCT)统计借阅人数
- 使用事务保证借阅和归还的原子性
