## 案例1：学生信息管理系统 - DDL实践

### 1.1 案例基本信息
- **案例名称**：学生信息管理系统 - 数据库和表的创建
- **对应讲义**：第一章 1.2.1节 DDL（数据定义语言）
- **难度等级**：⭐ 入门级
- **预计时间**：30分钟

### 1.2 展示目的
通过本案例，学员将学会：
1. 如何使用DDL语句创建数据库
2. 如何使用DDL语句创建表并定义约束
3. 如何查看数据库和表结构
4. 理解主键、自增、非空约束的作用

### 1.3 对应讲义内容
- **讲义位置**：第一章 1.2.1 DDL（数据定义语言）
- **对应代码**：
```sql
-- 创建数据库
CREATE DATABASE school;

-- 创建学生表
CREATE TABLE students (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    age INT,
    gender VARCHAR(10),
    birthday DATE
);
```

### 1.4 案例步骤

#### 步骤1：创建数据库
```sql
CREATE DATABASE school;
```

#### 步骤2：使用数据库
```sql
USE school;
```

#### 步骤3：创建学生表
```sql
CREATE TABLE students (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    age INT,
    gender VARCHAR(10),
    birthday DATE
);
```

#### 步骤4：查看表结构
```sql
DESC students;
```

### 1.5 测试方法

#### 测试1：验证数据库创建成功
**输入**：
```sql
SHOW DATABASES;
```

**预期输出解读**：
```
+--------------------+
| Database           |
+--------------------+
| information_schema |
| mysql              |
| performance_schema |
| school             |  ← 这里应该出现school数据库
| sys                |
+--------------------+
```
**解读说明**：
- 讲义中提到：CREATE DATABASE用于创建数据库
- 输出中出现`school`表示数据库创建成功
- 学员应理解：`SHOW DATABASES`命令用于查看所有数据库

#### 测试2：验证表结构正确
**输入**：
```sql
DESC students;
```

**预期输出解读**：
```
+----------+-------------+------+-----+---------+----------------+
| Field    | Type        | Null | Key | Default | Extra          |
+----------+-------------+------+-----+---------+----------------+
| id       | int(11)     | NO   | PRI | NULL    | auto_increment |
| name     | varchar(50) | NO   |     | NULL    |                |
| age      | int(11)     | YES  |     | NULL    |                |
| gender   | varchar(10) | YES  |     | NULL    |                |
| birthday | date        | YES  |     | NULL    |                |
+----------+-------------+------+-----+---------+----------------+
```
**解读说明**：
- 讲义中详细解释了各列的含义，学员应能对照理解：
  - `Field`：字段名
  - `Type`：数据类型
  - `Null`：是否允许为空（NO表示NOT NULL约束生效）
  - `Key`：键类型（PRI表示PRIMARY KEY主键）
  - `Extra`：额外信息（auto_increment表示自增）
- 学员应理解：`id`字段的`NO`在Null列对应`NOT NULL`约束，`PRI`对应`PRIMARY KEY`约束，`auto_increment`对应`AUTO_INCREMENT`特性

#### 测试3：测试主键和自增功能
**输入**：
```sql
INSERT INTO students (name, age, gender) VALUES ('张三', 20, '男');
INSERT INTO students (name, age, gender) VALUES ('李四', 21, '女');
SELECT * FROM students;
```

**预期输出解读**：
```
+----+--------+-----+--------+----------+
| id | name   | age | gender | birthday |
+----+--------+-----+--------+----------+
| 1  | 张三   | 20  | 男     | NULL     |
| 2  | 李四   | 21  | 女     | NULL     |
+----+--------+-----+--------+----------+
```
**解读说明**：
- 讲义中提到：`AUTO_INCREMENT`使id字段自动递增
- 输出中`id`字段自动从1开始递增（1、2...）
- 学员应理解：主键`id`无需手动指定，系统自动分配唯一值
- 讲义中强调：主键约束保证每条记录的`id`都是唯一的

#### 测试4：测试NOT NULL约束
**输入**：
```sql
INSERT INTO students (age, gender) VALUES (22, '男');
```

**预期输出解读**：
```
ERROR 1364 (HY000): Field 'name' doesn't have a default value
```
**解读说明**：
- 讲义中说明：`NOT NULL`约束表示该字段不能为空
- 报错信息说明`name`字段设置了`NOT NULL`约束，必须提供值
- 学员应理解：约束的作用是在数据插入时进行检查，保证数据完整性

### 1.6 常见问题与解答

**Q1：为什么id字段不需要手动插入值？**
A：讲义中解释了`AUTO_INCREMENT`特性，该特性使id字段自动递增，无需手动指定。

**Q2：为什么name字段必须插入值？**
A：讲义中说明了`NOT NULL`约束，该约束要求该字段必须有值，不能为NULL。

**Q3：如何修改表结构？**
A：讲义1.2.1节中介绍了ALTER TABLE语句，可以添加、修改、删除列。

## 案例2：学生数据管理 - DML实践

### 2.1 案例基本信息
- **案例名称**：学生数据管理 - 数据的增删改查
- **对应讲义**：第一章 1.2.2节 DML（数据操作语言）
- **难度等级**：⭐⭐ 基础级
- **预计时间**：45分钟

### 2.2 展示目的
通过本案例，学员将学会：
1. 如何使用INSERT语句插入数据
2. 如何使用UPDATE语句更新数据
3. 如何使用DELETE语句删除数据
4. 如何使用SELECT语句查询数据
5. 理解WHERE条件的重要性

### 2.3 对应讲义内容
- **讲义位置**：第一章 1.2.2 DML（数据操作语言）
- **对应代码**：
```sql
-- 插入数据
INSERT INTO students (name, age, gender) VALUES ('张三', 20, '男');

-- 更新数据
UPDATE students SET age = 21 WHERE name = '张三';

-- 删除数据
DELETE FROM students WHERE id = 1;

-- 查询数据
SELECT * FROM students;
```

### 2.4 案例步骤

#### 步骤1：插入多条学生数据
```sql
INSERT INTO students (name, age, gender, birthday) VALUES

('赵六', 22, '女', '2001-03-25');
```

#### 步骤2：查询所有学生
```sql
SELECT * FROM students;
```

#### 步骤3：更新特定学生的年龄
```sql
UPDATE students SET age = 21 WHERE name = '张三';
```

#### 步骤4：删除特定学生
```sql
DELETE FROM students WHERE name = '王五';
```

#### 步骤5：查询删除后的结果

### 2.6 常见问题与解答

**Q1：为什么不带WHERE条件的UPDATE很危险？**
A：讲义中警告：不带WHERE条件会更新表中所有记录，可能导致数据错误。学员必须养成使用WHERE条件的习惯。

**Q2：DELETE和DROP有什么区别？**
A：讲义中说明：DELETE删除表中的数据，保留表结构；DROP删除整个表（包括表结构和数据）。

**Q3：如何防止误操作？**
A：建议在执行UPDATE或DELETE前，先用SELECT查询确认要操作的记录，确认无误后再执行更新或删除。

## 案例3：学生成绩查询 - DQL复杂查询实践

### 3.1 案例基本信息
- **案例名称**：学生成绩查询 - 复杂条件查询、排序、聚合
- **对应讲义**：第一章 1.2.3节 DQL（数据查询语言）
- **难度等级**：⭐⭐⭐ 进阶级
- **预计时间**：60分钟

### 3.2 展示目的
通过本案例，学员将学会：
1. 如何使用WHERE子句进行条件查询
2. 如何使用ORDER BY进行排序
3. 如何使用聚合函数进行数据统计
4. 如何使用GROUP BY进行分组
5. 如何使用HAVING进行分组筛选
6. 如何进行多表连接查询

### 3.3 对应讲义内容
- **讲义位置**：第一章 1.2.3 DQL（数据查询语言）
- **对应代码**：
```sql
-- 条件查询
SELECT * FROM students WHERE age > 20;

-- 排序查询
SELECT * FROM students ORDER BY age DESC;

-- 聚合函数
SELECT COUNT(*) FROM students;
SELECT AVG(age) FROM students;

-- 分组查询
SELECT gender, COUNT(*) FROM students GROUP BY gender;

-- 连接查询
SELECT s.name, c.course_name 
FROM students s INNER JOIN courses c ON s.id = c.student_id;
```

### 3.4 案例准备：创建课程表和成绩表

#### 步骤1：创建课程表
```sql
CREATE TABLE courses (
    id INT PRIMARY KEY AUTO_INCREMENT,
    course_name VARCHAR(50) NOT NULL,
    teacher VARCHAR(50)
);
```

#### 步骤2：创建成绩表
```sql
CREATE TABLE scores (
    id INT PRIMARY KEY AUTO_INCREMENT,
    student_id INT,
    course_id INT,
    score DECIMAL(5,2),
    FOREIGN KEY (student_id) REFERENCES students(id),
    FOREIGN KEY (course_id) REFERENCES courses(id)
);
```

#### 步骤3：插入课程数据
```sql
INSERT INTO courses (course_name, teacher) VALUES
('数学', '王老师'),
('英语', '李老师'),
('语文', '张老师'),
('物理', '赵老师');
```

#### 步骤4：插入成绩数据
```sql
INSERT INTO scores (student_id, course_id, score) VALUES
(1, 1, 85.5),  -- 张三 数学 85.5
(1, 2, 90.0),  -- 张三 英语 90.0
(1, 3, 78.5),  -- 张三 语文 78.5
(2, 1, 92.0),  -- 李四 数学 92.0
(2, 2, 88.5),  -- 李四 英语 88.5
(2, 4, 95.0);  -- 李四 物理 95.0
```

### 3.5 案例步骤

#### 步骤1：基础条件查询
```sql
-- 查询年龄大于20岁的学生
SELECT * FROM students WHERE age > 20;

-- 查询性别为女的学生
SELECT * FROM students WHERE gender = '女';

-- 查询年龄在20到22岁之间的学生
SELECT * FROM students WHERE age BETWEEN 20 AND 22;

-- 查询姓张的学生
SELECT * FROM students WHERE name LIKE '张%';
```

#### 步骤2：排序查询
```sql
-- 按年龄升序排序
SELECT * FROM students ORDER BY age ASC;

-- 按年龄降序排序
SELECT * FROM students ORDER BY age DESC;

-- 先按性别排序，再按年龄排序
SELECT * FROM students ORDER BY gender ASC, age DESC;
```

#### 步骤3：聚合函数查询
```sql
-- 统计学生总数
SELECT COUNT(*) AS 学生总数 FROM students;

-- 计算平均年龄
SELECT AVG(age) AS 平均年龄 FROM students;

-- 查找最大年龄
SELECT MAX(age) AS 最大年龄 FROM students;

-- 查找最小年龄
SELECT MIN(age) AS 最小年龄 FROM students;

-- 统计总成绩
SELECT SUM(score) AS 总成绩 FROM scores;
```

#### 步骤4：分组查询
```sql
-- 按性别统计学生人数
SELECT gender, COUNT(*) AS 人数 
FROM students 
GROUP BY gender;

-- 按课程统计平均分
SELECT c.course_name, AVG(s.score) AS 平均分
FROM courses c
INNER JOIN scores s ON c.id = s.course_id
GROUP BY c.course_name;

-- 统计每个学生的总成绩
SELECT st.name, SUM(sc.score) AS 总成绩
FROM students st
INNER JOIN scores sc ON st.id = sc.student_id
GROUP BY st.name;
```

#### 步骤5：HAVING分组筛选
```sql
-- 查询平均分大于85分的课程
SELECT c.course_name, AVG(s.score) AS 平均分
FROM courses c
INNER JOIN scores s ON c.id = s.course_id
GROUP BY c.course_name
HAVING AVG(s.score) > 85;

-- 查询总成绩超过170分的学生
SELECT st.name, SUM(sc.score) AS 总成绩
FROM students st
INNER JOIN scores sc ON st.id = sc.student_id
GROUP BY st.name
HAVING SUM(sc.score) > 170;
```

#### 步骤6：多表连接查询
```sql
-- 查询所有学生的成绩信息（内连接）
SELECT 
    st.name AS 学生姓名,
    c.course_name AS 课程名称,
    sc.score AS 成绩
FROM students st
INNER JOIN scores sc ON st.id = sc.student_id
INNER JOIN courses c ON sc.course_id = c.id;

-- 查询所有学生及其课程（左连接，包含没有成绩的学生）
SELECT 
    st.name AS 学生姓名,
    c.course_name AS 课程名称,
    sc.score AS 成绩
FROM students st
LEFT JOIN scores sc ON st.id = sc.student_id
LEFT JOIN courses c ON sc.course_id = c.id;
```

### 3.6 测试方法与结果解读

#### 测试1：基础条件查询
**输入**：
```sql
-- 查询年龄大于20岁的学生
SELECT * FROM students WHERE age > 20;
```

**预期输出解读**：
```
+----+--------+-----+--------+------------+
| id | name   | age | gender | birthday   |
+----+--------+-----+--------+------------+
| 1  | 张三   | 21  | 男     | 2003-05-15 |
| 2  | 李四   | 21  | 女     | 2002-08-20 |
| 4  | 赵六   | 22  | 女     | 2001-03-25 |
+----+--------+-----+--------+------------+
```
**解读说明**：
- 讲义中说明：WHERE子句用于筛选符合条件的记录
- 学员应理解：
  - `>`是大于运算符
  - `age > 20`表示筛选年龄字段大于20的记录
  - 输出中所有学生的年龄都大于20（21、21、22）
- 讲义中强调：WHERE子句是查询中最常用的功能

**输入**：
```sql
-- 查询年龄在20到22岁之间的学生
SELECT * FROM students WHERE age BETWEEN 20 AND 22;
```

**预期输出解读**：
```
+----+--------+-----+--------+------------+
| id | name   | age | gender | birthday   |
+----+--------+-----+--------+------------+
| 1  | 张三   | 21  | 男     | 2003-05-15 |
| 2  | 李四   | 21  | 女     | 2002-08-20 |
| 4  | 赵六   | 22  | 女     | 2001-03-25 |
```
**解读说明**：
- 讲义中介绍：BETWEEN运算符用于范围查询
- 学员应理解：
  - `BETWEEN 20 AND 22`表示年龄在20到22之间（包含边界值）
  - 相当于`age >= 20 AND age <= 22`
  - 输出中学生的年龄分别是21、21、22，都在范围内

**输入**：
```sql
-- 查询姓张的学生
SELECT * FROM students WHERE name LIKE '张%';
```

**预期输出解读**：
```
+----+--------+-----+--------+------------+
| id | name   | age | gender | birthday   |
+----+--------+-----+--------+------------+
| 1  | 张三   | 21  | 男     | 2003-05-15 |
+----+--------+-----+--------+------------+
```
**解读说明**：
- 讲义中介绍：LIKE运算符用于模糊查询
- 学员应理解：
  - `%`是通配符，表示任意多个字符
  - `'张%'`表示以"张"开头的字符串
  - 输出中只返回姓名为"张三"的记录

```sql
FROM students

```

**输入**：
```sql
-- 统计每个学生的总成绩
SELECT st.name, SUM(sc.score) AS 总成绩
FROM students st
INNER JOIN scores sc ON st.id = sc.student_id
GROUP BY st.name;
```

**预期输出解读**：
```
+--------+----------+
| name   | 总成绩   |
+--------+----------+
| 李四   |   275.50 |

+--------+----------+
```

- 学员应理解：

  - 按学生姓名（st.name）分组
  - SUM(sc.score)计算每个学生的总成绩

  - 李四的总成绩 = 92.0 + 88.5 + 95.0 = 275.5
- 讲义中说明：分组查询是数据统计分析的重要工具

#### 测试5：HAVING分组筛选
**输入**：
```sql
-- 查询平均分大于85分的课程
SELECT c.course_name, AVG(s.score) AS 平均分
FROM courses c
INNER JOIN scores s ON c.id = s.course_id
GROUP BY c.course_name
HAVING AVG(s.score) > 85;
```

**预期输出解读**：
```
+-------------+----------+
| course_name | 平均分   |
+-------------+----------+
| 数学        |   88.75  |
| 英语        |   89.25  |
+-------------+----------+
```
**解读说明**：
- 讲义中强调：HAVING用于对分组后的结果进行筛选
- 学员应理解：
  - GROUP BY按课程分组
  - AVG(s.score)计算每门课程的平均分
  - HAVING筛选平均分大于85的课程
  - 数学平均分 = (85.5 + 92.0) / 2 = 88.75 > 85 ✓
  - 英语平均分 = (90.0 + 88.5) / 2 = 89.25 > 85 ✓
  - 语文平均分 = 78.5 < 85 ✗（不满足条件）
  - 物理平均分 = 95.0 > 85 ✓（但物理只有一个人，不符合条件）
- 讲义中重要提示：HAVING和WHERE的区别
  - WHERE在分组前筛选记录
  - HAVING在分组后筛选组

**关键区别讲解**（讲义重点）：
```sql
-- WHERE：在分组前筛选
-- 示例：查询年龄大于20岁的学生，按性别分组
SELECT gender, COUNT(*) 
FROM students 
WHERE age > 20  -- 先筛选年龄大于20的学生
GROUP BY gender;

-- HAVING：在分组后筛选
-- 示例：查询人数大于1的性别组
SELECT gender, COUNT(*) 
FROM students 
GROUP BY gender
HAVING COUNT(*) > 1;  -- 先分组，再筛选人数大于1的组
```

#### 测试6：多表连接查询
**输入**：
```sql
-- 查询所有学生的成绩信息（内连接）
SELECT 
    st.name AS 学生姓名,
    c.course_name AS 课程名称,
    sc.score AS 成绩
FROM students st
INNER JOIN scores sc ON st.id = sc.student_id
INNER JOIN courses c ON sc.course_id = c.id;
```

**预期输出解读**：
```
+----------+----------+-------+
| 学生姓名 | 课程名称 | 成绩  |
+----------+----------+-------+
| 张三     | 数学     |  85.5 |
| 张三     | 英语     |  90.0 |
| 张三     | 语文     |  78.5 |
| 李四     | 数学     |  92.0 |
| 李四     | 英语     |  88.5 |
| 李四     | 物理     |  95.0 |
+----------+----------+-------+
```
**解读说明**：
- 讲义中介绍：INNER JOIN（内连接）只返回两个表中匹配的记录
- 学员应理解：
  - 第一个INNER JOIN：连接students和scores表，条件是`st.id = sc.student_id`
  - 第二个INNER JOIN：连接scores和courses表，条件是`sc.course_id = c.id`
  - 只返回有成绩记录的学生和课程
  - 输出显示张三有3门课成绩，李四有3门课成绩
- 讲义中说明：连接查询是SQL中最强大的功能之一

**输入**：
```sql
-- 查询所有学生及其课程（左连接，包含没有成绩的学生）
-- 先插入一个没有成绩的学生
INSERT INTO students (name, age, gender, birthday) 
VALUES ('钱七', 20, '男', '2003-01-01');

SELECT 
    st.name AS 学生姓名,
    c.course_name AS 课程名称,
    sc.score AS 成绩
FROM students st
LEFT JOIN scores sc ON st.id = sc.student_id
LEFT JOIN courses c ON sc.course_id = c.id
ORDER BY st.name;
```

**预期输出解读**：
```
+----------+----------+-------+
| 学生姓名 | 课程名称 | 成绩  |
+----------+----------+-------+
| 张三     | 数学     |  85.5 |
| 张三     | 英语     |  90.0 |
| 张三     | 语文     |  78.5 |
| 李四     | 数学     |  92.0 |
| 李四     | 英语     |  88.5 |
| 李四     | 物理     |  95.0 |
| 钱七     | NULL     |  NULL |
+----------+----------+-------+
```
**解读说明**：
- 讲义中介绍：LEFT JOIN（左连接）返回左表所有记录，右表没有匹配的显示NULL
- 学员应理解：
  - LEFT JOIN从students表（左表）开始
  - 钱七虽然插入到students表，但没有成绩记录
  - 由于使用LEFT JOIN，钱七仍然出现在结果中
  - 钱七的`课程名称`和`成绩`都是NULL
- 讲义中强调：内连接 vs 外连接的区别
  - INNER JOIN：只返回匹配的记录
  - LEFT JOIN：返回左表所有记录
  - RIGHT JOIN：返回右表所有记录

### 3.7 常见问题与解答

**Q1：WHERE和HAVING有什么区别？**
A：讲义中详细说明：
- WHERE：在分组前筛选记录，不能使用聚合函数
- HAVING：在分组后筛选组，可以使用聚合函数

**Q2：INNER JOIN和LEFT JOIN有什么区别？**
A：讲义中介绍：
- INNER JOIN（内连接）：只返回两个表中匹配的记录
- LEFT JOIN（左连接）：返回左表所有记录，右表没有匹配的显示NULL

**Q3：为什么有些课程的平均分不在结果中？**
A：因为使用HAVING筛选了平均分大于85的课程，不满足条件的课程不会出现在结果中。

## 案例4：用户权限管理 - DCL实践

### 4.2 展示目的
通过本案例，学员将学会：
1. 如何创建MySQL用户

### 4.3 对应讲义内容
- **讲义位置**：第一章 1.2.4 DCL（数据控制语言）
- **对应代码**：

### 4.4 案例步骤


```sql
-- 创建可以查询和插入的用户
CREATE USER 'readwrite_user'@'localhost' IDENTIFIED BY 'write123';

GRANT SELECT, INSERT ON school.* TO 'readwrite_user'@'localhost';

-- 刷新权限
FLUSH PRIVILEGES;
```

#### 步骤3：查看用户权限
```sql
-- 查看只读用户权限
SHOW GRANTS FOR 'readonly_user'@'localhost';

-- 查看读写用户权限
SHOW GRANTS FOR 'readwrite_user'@'localhost';
```

#### 步骤4：测试权限
```sql

-- 尝试插入（应该失败）

```

#### 步骤5：修改和撤销权限
```sql
-- 授予读写用户更新权限
GRANT UPDATE ON school.* TO 'readwrite_user'@'localhost';

-- 撤销只读用户的查询权限（谨慎操作）
REVOKE SELECT ON school.* FROM 'readonly_user'@'localhost';

-- 刷新权限
FLUSH PRIVILEGES;
```

#### 步骤6：删除用户
```sql

DROP USER 'readonly_user'@'localhost';

```

### 4.6 常见问题与解答

**Q1：什么是最小权限原则？**
A：讲义中解释：最小权限原则是指只授予用户完成任务所需的最小权限，避免授予过大的权限，降低安全风险。

**Q2：为什么要FLUSH PRIVILEGES？**
A：讲义中说明：FLUSH PRIVILEGES用于刷新权限，使权限更改立即生效。在某些情况下，授权后会自动刷新，但建议显式执行以确保生效。

**Q3：如何授予所有权限？**
A：使用`GRANT ALL PRIVILEGES ON database.* TO 'user'@'host'`，但讲义中强调：尽量避免授予所有权限，应遵循最小权限原则。

#### 测试2：验证InnoDB事务支持（关键测试）
**输入**：
```sql
-- 开始事务
START TRANSACTION;

-- 插入数据
INSERT INTO test_innodb (name, balance) VALUES ('张三', 1000);
INSERT INTO test_innodb (name, balance) VALUES ('李四', 2000);

-- 查询结果
SELECT * FROM test_innodb;

-- 回滚事务
ROLLBACK;

-- 再次查询

```

**事务内查询输出**：
```
+----+--------+---------+
| id | name   | balance |
+----+--------+---------+
| 1  | 张三   | 1000.00 |
| 2  | 李四   | 2000.00 |
+----+--------+---------+
```

**回滚后查询输出**：
```

```
**解读说明**：
- ⭐ 讲义中重点说明：InnoDB支持ACID事务特性
- 学员应理解：
  - `START TRANSACTION`开始一个事务
  - 在事务内可以执行多个SQL语句
  - `ROLLBACK`回滚事务，撤销所有操作
  - 回滚后，表中的数据为空，说明事务回滚成功
- 讲义中强调：事务的四个特性（ACID）
  - 原子性（Atomicity）：事务中的操作要么全部成功，要么全部失败
  - 一致性（Consistency）：事务执行前后，数据库保持一致状态
  - 隔离性（Isolation）：多个事务并发执行时，互不干扰
  - 持久性（Durability）：事务提交后，数据永久保存

**事务提交测试**：
**输入**：
```sql
-- 开始事务
START TRANSACTION;

INSERT INTO test_innodb (name, balance) VALUES ('张三', 1000);
INSERT INTO test_innodb (name, balance) VALUES ('李四', 2000);

-- 提交事务
COMMIT;

-- 查询结果
SELECT * FROM test_innodb;
```

**提交后查询输出**：
```
+----+--------+---------+
| id | name   | balance |
+----+--------+---------+
| 1  | 张三   | 1000.00 |
| 2  | 李四   | 2000.00 |
+----+--------+---------+
```
**解读说明**：
- 讲义中说明：`COMMIT`提交事务，使所有操作永久生效
- 学员应理解：
  - 使用`COMMIT`而不是`ROLLBACK`
  - 提交后，数据被永久保存到数据库
  - 即使系统崩溃，数据也不会丢失
- 讲义中强调：在实际应用中，涉及资金、订单等重要数据时，必须使用事务保证数据一致性

| 特性 | InnoDB | MyISAM |
|------|--------|--------|
| 事务支持 | ✅ 支持 | ❌ 不支持 |
| 外键约束 | ✅ 支持 | ❌ 不支持 |
| 锁机制 | 行级锁 | 表级锁 |
| 读取性能 | 较好 | 非常好 |
| 写入性能 | 较好 | 一般 |
| 崩溃恢复 | ✅ 支持 | ❌ 不支持 |
| 适用场景 | 高并发OLTP、电商、金融 | 数据仓库、日志、以读为主 |

### 5.7 常见问题与解答

**Q1：为什么默认存储引擎是InnoDB？**
A：讲义中说明：MySQL 5.5+版本默认使用InnoDB，因为它支持事务和外键，适合大多数应用场景。

**Q2：行级锁和表级锁有什么区别？**
A：讲义中介绍：
- 行级锁：只锁定被访问的行，并发度高
- 表级锁：锁定整个表，并发度低
- InnoDB使用行级锁，适合高并发场景

**Q3：如何选择存储引擎？**
A：讲义中建议：
- 需要事务、外键、高并发：选择InnoDB
- 以读为主、大量数据、数据仓库：选择MyISAM
- 临时数据：选择Memory

**Q1：为什么要使用参数化查询？**
A：讲义中强调：参数化查询可以防止SQL注入攻击，提高程序安全性。永远不要使用字符串拼接构建SQL语句。

**Q2：为什么要commit？**
A：讲义中说明：commit用于提交事务，使INSERT、UPDATE、DELETE操作永久生效。InnoDB默认开启事务，不commit的话，修改不会保存。

**Q3：如何防止连接泄漏？**
A：讲义中建议：使用try-finally确保连接关闭，或者使用with语句自动管理连接资源。
